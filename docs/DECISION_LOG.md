## 📅 로그 (Log)

### [2026-04-27] Firebase 스타트업 크레딧 적용 방식 확인
* **이슈**: 신규 Mozzy II 프로젝트에 기존(bling_app)의 Google for Startups Cloud Program 크레딧을 연동할 수 있는지 확인.
* **결정/팩트**: 스타트업 크레딧은 프로젝트 ID가 아니라 **결제 계정(Billing Account)** 단위로 부여됨. 따라서 신규 파이어베이스 프로젝트 생성 후 Blaze 요금제로 업그레이드할 때 기존 결제 계정을 연결하면 크레딧이 동일하게 적용됨.
* **조치**: 파이어베이스 셋업 시 프로젝트 생성 -> Blaze 업그레이드 -> 기존 결제 계정 연결 순서로 진행하기로 함.

### [2026-04-27] Firestore 132개 인덱스 제한 및 국가 코드 전략
* **이슈**: V1(bling_app)에서 인덱스가 350개로 팽창했던 문제 해결 및 Global One Build 대비.
* **결정**: `countryCode`를 쿼리 필드에 포함시켜 인덱스를 복사 생성하는 기존 방식을 폐기. 대신 컬렉션 상위 경로(Path)를 국가 코드로 분리(Sharding)하여 물리적으로 데이터를 격리하는 **경로 기반 격리(Path-based Sharding)** 도입.
* **조치**: `docs/Firestore_Architecture_Strategy.md` 문서 발행 완료. Phase 0의 `MozzyPostContract`에 관련 사상(geoPath 직렬화 등) 적용 완료.
